# API 接入指南

## 接入流程概述

接入本平台 API 分为四步：注册账号、创建应用、获取 API Key、调用接口。完成后即可在生产环境调用实名认证、企业信息查询、风控评分等接口。

## 第一步：注册与实名

在控制台注册企业账号并完成企业实名认证。实名通过后才能创建生产应用和获取正式 API Key。

## 第二步：创建应用

进入控制台「应用管理」，创建应用并选择需要开通的 API 产品。每个应用拥有独立的 API Key 与权限配置。一个企业可创建多个应用，用于区分不同业务线。

## 第三步：获取 API Key

应用创建后，在应用详情页获取 API Key 与 Secret Key。API Key 用于标识调用方身份，Secret Key 用于请求签名，**Secret Key 仅在创建时展示一次，请妥善保存**。如遗失需重新生成。

## 第四步：调用接口

所有接口均通过 HTTPS POST 调用，请求需携带 API Key 并按《鉴权与签名》文档计算签名。基本请求结构：

```
POST https://api.example.com/v1/idcard/verify
Headers:
  X-Api-Key: 你的API Key
  X-Timestamp: 当前秒级时间戳
  X-Sign: 按规则计算的签名
Body (JSON): 业务参数
```

## 调用前检查清单

- 应用已开通目标 API 产品（否则返回 PRODUCT_NOT_ENABLED）。
- API Key 处于有效状态且未过期（否则返回 AUTH_KEY_EXPIRED）。
- 服务器出口 IP 已加入白名单（如开启白名单，否则返回 IP_NOT_ALLOWED）。
- 套餐有足够的调用量与余额。

## 环境与超时建议

建议客户端请求超时设置不低于 5 秒，并实现失败重试与指数退避。生产接入前建议先在测试用例中覆盖鉴权失败、参数错误、限流等异常分支。
