# Webhook 回调

## 回调机制

对于异步类接口（如批量核验、风控异步评分），平台在处理完成后通过 Webhook 主动向客户配置的回调地址推送结果。回调为 HTTPS POST，body 为 JSON，携带 `event_id`、业务结果与签名。

## 回调地址配置

在控制台「应用管理 - 回调配置」中设置回调 URL。回调地址必须公网可访问、支持 HTTPS，并能在 5 秒内返回 HTTP 200。

## 回调成功的判定

客户服务端收到回调后，**必须返回 HTTP 200** 表示已接收。返回非 2xx、超时或不可达，平台均视为投递失败（WEBHOOK_DELIVERY_FAILED）。

## 回调重试策略

投递失败后，平台按指数退避自动重试，最多重试 6 次，间隔依次约为 1 分钟、5 分钟、30 分钟、2 小时、6 小时、12 小时。超过最大重试次数后停止，可在控制台手动触发补发。

## 回调收不到的排查

1. 确认回调地址在控制台已正确配置且回调功能已启用。
2. 确认回调地址公网可达（可用 curl 从外网测试），未被防火墙/安全组拦截。
3. 查看投递记录：若显示客户返回 500 或超时，说明平台已发送但客户服务处理失败。
4. 区分四种情况：平台未发送、客户未接收、客户返回错误、验签失败。

## 回调验签（WEBHOOK_SIGN_INVALID）

回调 body 携带签名，客户需使用回调密钥按文档验签，确认回调来自平台。验签失败常见于：使用了错误的密钥、对原始 body 做了改写或重新序列化导致字节不一致。建议对**原始请求体**验签，不要先反序列化再重新拼接。

## 回调重复

由于重试机制，客户可能收到重复回调。请使用 `event_id` 做幂等去重，确保同一事件只处理一次。
